These days JavaScript is everywhere. I started learning JavaScript in earnest at Xerox while developing control and UI software for the text fixtures I designed. At Integra I maintained and expanded their testing scripts that perform automated testing on their flagship product. These tests were written in JavaScript and executed through the Windows Script interpreter, wScript.

I have used JavaScript to create animated and Responsive webpages (like this one) for several years, but began to spend significant effort in 2013 expanding my knowledge of both front and back end libraries, as well as communication protocols like REST APIs and AJAX calls. There has been an explosion of cheap, networked, 32-bit mini-computers like the Raspberry Pi and Arduino, as well as the growing trend of the Internet of Things (IoT). I believe that JavaScript provides the tools to build awesome web-based user interfaces for IoT devices and server-side JavaScript (like Node.js) allows for control of those embedded devices too. I am currently playing with a Raspberry Pi to develop an embedded webserver that can remotely read sensors and actuate switches, all controlled with JavaScript.

Testing and Automation

At Xerox I wrote test scripts in JavaScript that would run functional tests on the circuit board sub-components of their printer products. As the circuit boards would reach the end of the assembly line, prior to final assembly, they would be put into a test fixture. A bed of needles would be lowered onto the circuit board and connect it to a computer on the test fixture. I designed the interface hardware, programmed the embedded microcontrollers in C++, and wrote the functional testing software on the computer in JavaScript. The software would put the circuit board through its paces and catalog any unexpected behavior. Tests were prioritized and results uploaded to a database on a central server for data mining. These fixtures were essential for Xerox's six-sigma management strategy and warranty insurance research.


My position at Integra was as a software tester. About half my job was manual testing and the other half I spent on maintaining their suite of automated testing scripts, written in JavaScript. During my time there, I added dozens of test cases, made several improvements to the utility libraries, and assisted in the design and build out of a Build Verification Test (BVT) program that automated the testing of the nightly build.

Back End: Node.js and Embedded Web Server

I became interested in Node (or Node.js) and the Express.js framework for its ability to execute Back-end/Server-side functions with programs written in JavaScript. This is handy in many places, but my personal interest was in open-source, embedded web servers such as the Raspberry Pi. For purposes of demonstration on this site, I ported the code to Node servers running Windows 2008 Server in the Amazon EC2 Cloud, then later moved the same code to Digital Ocean Droplets running Unbuntu Linux. Because the code is in JavaScript and Node runs on all these operating systems, no code changes were necessary.

The code examples never progressed past the prototyping stage, so they used simple CSV files for data storage. For scalability, a SQL database would normally be used, but CSV is great for small websites and quick prototyping. I am currently taking courses at Code School to practice building and maintaining SQL databases. Node.js has several excellent SQL API libraries.
