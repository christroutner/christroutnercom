Full Stack Applications

“Full stack” means a lot of different things to different people. In my mind, JavaScript embodies the idea of ‘full stack’ the best by creating a common programming language that seamlessly spans between the server and browser. At a minimum, a ‘full stack’ application incorporates a database for persisting data, API calls to pass data between the server and client browser, and a mobile-first user interface running in a client’s web browser. I've worked almost exclusively on full stack application since the beginning of 2015. Here are some examples:


  • ConnextCMS – One of my first tasks as lead developer at Skagit Connext was to find a free or open source Content Management System (CMS) that was not WordPress. We used Expression Engine CMS which comes with a $300 licensing fee. It was great for high-end web sites, but we needed something mobile-friendly, simple, and free for our smaller customers.

    I discovered KeystoneJS CMS, a Node/Express/MongoDB application. The core KeystoneJS engine is an elegant design. However, the Admin UI is written in React.js and uses server-side rendering. This moves the development onto the server, and I felt a better design is to keep the UI development in the browser. ConnextCMS is a Backbone.js application that mimics the WordPress Admin UI using the AdminLTE Control Panel Bootstrap Template. Data is exchanged between server and client using a REST API.

    As I developed ConnextCMS, I convinced the management to release the code as open source under the MIT license. We are currently in the process of trying to build momentum and recruit developers from the open source community. Visit ConnextCMS.com for more information and a list of features.

  • Pioneer Ads – This is an extremely complex web form on par with the kind of forms/web apps used by TurboTax or MyCorporation. This form is used internally by Pioneer News Group to coordinate online advertising between its child companies. The code is not publically available. The form runs on top of KeystoneJS, similar to ConnextCMS. It has some of the following features:

    • Data is saved in real time as the form is filled out.

    • If an error occurs due to browser or OS incompatibility, a log file is emailed to the administrator.

    • Image and video uploads are validated against size and dimension constraints prior to upload.

    • Using jQuery animation, a slide-out contextual-help screen automatically scrolls to the help you need based on your location within the form.

    • Various parts of the form are hidden or shown based on decisions made higher up in the form.

    • Old orders have the ability to be cloned and reordered with a couple clicks. Unfinished orders can be saved and resumed at a later date. Reminder emails automatically get sent on a weekly basis to people with unsubmitted orders.

    • A custom, back end dashboard allows administrators to easily review and process orders.


  • RPiOVN is an acronym for Raspberry Pi Open Value Network. It's a cooperative for programmers of the Raspberry Pi minicomputer who use it to develop new applications and appliances. The RPiOVN dashboard pictured here is used for members of the cooperative to track the amount of time they've contributed toward a project. Hours are accounted and profit is distributed fairly. All time tracked is transparent to all members.

    If that sounds complicated, it is. You should probably just create an account and check it out for yourself.

    I created RPiOVN.org after working with similar software used by Sensorica and studying the phenomenon of Open Value Networks. It's a sophisticated web application that does on-the-fly accounting and data visualization. Users can easily enter information about work they've done inside the /logwork dashboard. Work entries are stored in a MongodDB database. Records can be viewed and even downloaded in Excel format by members. I've published all code as open source on GitHub. The software is packaged as a plugin for ConnextCMS.

    • View GitHub Repository


  • CrumbShare is both a web app and an embedded device. Crumb Share is a service that works in conjunction with the RPi-Tracker, a small tracking appliance. The business targets travel writers, travel bloggers, and people who adventure in boats and vehicles, who want to easily share a breadcrumb trail of their travels on social media.

    Both the Crumb Share service and the RPi-Tracker device are spin-off products of RPi OVN. I created the device and started the service with a friend and fellow website designer. ConnextCMS is used to run the site as well as handle the upload of data from devices scattered across the globe.

    The devices automatically sync up with the server any time they come within reach of an internet WiFi signal. The users breadcrumb trail is displayed on top of a Google map. Users get a link they can use to share their map on social media, or a simple line of code they can use to embed it on their website.

    Privacy features allow users to draw keep-out circles around sensitive locations where the devices will not track them. Control of the device takes place over WiFi and can be done with a smart phone or laptop. Everything is programmed in JavaScript.

    Create an account and check it out for yourself!

Whenever a dashboard is called for, my borrow heavily from the open source AdminLTE Template, also based on Bootstrap. I also have experience with several JavaScript frameworks, as explained below.


  • Backbone.js & Require.js

    All the apps I've listed under Full Stack Development have been based on Bootstrap.js. I also modularize my projects with Require.js. When collaborating on projects, I frequently reference this open source book by O'Reily media. It's concise, freely available, and helps other developers access the knowledge they need to copy the code I've developed and released as open source.

    Although Backone.js is no longer as in-vogue as React.js or Angular.js, I firmly believe that it is the superior framework due to its small download size, unopinionated architecture, and maturity. Backbone.js was battle tested and code mature before Angular or React appears on the scene. I believe these factors make it the lowest business risk for any organization, and I can do anythign with this framework that can be done with the others.

Back End: Node.js, IoT, and Embedded Web Server

I became interested in Node (or Node.js) and the Express.js framework for its ability to execute Back-end/Server-side functions with programs written in JavaScript. This is handy in many places, but my personal interest was in open-source, embedded web servers such as the Raspberry Pi. For purposes of demonstration on this site, I ported the code to Node servers running Windows 2008 Server in the Amazon EC2 Cloud, then later moved the same code to Digital Ocean Droplets running Ubuntu Linux. Because the code is in JavaScript and Node runs on all these operating systems, no code changes were necessary.


  • SPOT Tracker Map - The server linked to my SPOT satellite tracker data is published as an XML file. Only the last 50 data points are available. As new data comes in, old data is discarded, but I never use more than 50 data points in a day. I wrote an application in Node that downloads the XML file and parses the data into a CSV file. This data is then retrieved by the client-side JavaScript running in the Client's browser.

  • Collaborative Geo-Data - This is a prototype Google Maps web app that allows users to collaboratively update a common database of geographical data. I had intended to use this application to allow other boaters to share information about anchorages in remote locations and sources of wild food nearby. These forms make POST and GET requests to a custom web-service written in JavaScript and running in Node. These services update the CSV file, allowing for collaborative editing of the database.

    The code examples never progressed past the prototyping stage, so it used simple CSV files for data storage. For scalability, a database would normally be used, but CSV is great for small websites and quick prototyping.

Data can easily be downloaded and uploaded to Google Maps. The intended audience is boaters and RVers who want a cheap, dedicated tracking device for sharing their travels.

Testing and Automation

At Xerox I wrote test scripts in JavaScript that would run functional tests on the circuit board sub-components of their printer products. As the circuit boards would reach the end of the assembly line, prior to final assembly, they would be put into a test fixture. A bed of needles would be lowered onto the circuit board and connect it to a computer on the test fixture. I designed the interface hardware, programmed the embedded microcontrollers in C++, and wrote the functional testing software on the computer in JavaScript. The software would put the circuit board through its paces and catalog any unexpected behavior. Tests were prioritized and results uploaded to a database on a central server for data mining. These fixtures were essential for Xerox's six-sigma management strategy and warranty insurance research.


My position at Integra was as a software tester. About half my job was manual testing and the other half I spent on maintaining their suite of automated testing scripts, written in JavaScript. During my time there, I added dozens of test cases, made several improvements to the utility libraries, and assisted in the design and build out of a Build Verification Test (BVT) program that automated the testing of the nightly build.
